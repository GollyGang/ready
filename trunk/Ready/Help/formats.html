File Formats

 
Extended VTK image format: *.vti

The only file format currently supported by Ready is an extended version of VTK's image format, which uses XML. Below is an example: 

<?xml version="1.0"?>
<VTKFile type="ImageData" version="0.1" byte_order="LittleEndian" compressor="vtkZLibDataCompressor">
  <RD>
    <rule type="formula" name="Schlogl" number_of_chemicals="1" timestep="0.1">
      <formula>
        delta_a = laplacian_a + a - a*a*a;
      </formula>
    </rule>
  </RD>
  <ImageData WholeExtent="0 127 0 63 0 0" Origin="0 0 0" Spacing="1 1 1">
    <Piece Extent="0 127 0 63 0 0">
      <PointData Scalars="Scalars_">
        <DataArray type="Float32" Name="Scalars_" format="appended" RangeMin="-0.0010000000475" RangeMax="0.00099975580815" offset="0" />
      </PointData>
      <CellData>
      </CellData>
    </Piece>
  </ImageData>
  <AppendedData encoding="base64">
   _AQAAAACAAAAAAAAAqXAAAA==eJwkXHtcjPn3n6SbS5KQJGmTJBlJPedsMpIUUhQVRRGSJEmSth1t2 ... lots of data ... 3nudNY=
  </AppendedData>
</VTKFile>

This is the standard VTK format but with an RD element (inside the VTKFile element) that contains all the Ready-specific data. Such files can still be read as standard VTK image format, for example in Paraview.

<RD><rule ...This block describes the way the RD system operates. (required)
<RD><rule type="..."Can take two values. If type="formula" then the rule is specified by a differential equation in the formula element below. If type="inbuilt" then the rule is implemented in Ready itself. (required)
<RD><rule name="..."Contains the name of the rule. If type="inbuilt" then this must exactly match the name of one of Ready's inbuilt implementations. (required)
<RD><pattern_description ...This block describes the current image. (optional)
