File Formats

 
Extended VTK image format: *.vti

The only file format currently supported by Ready is an extended version of VTK's image format, which uses XML. Below is an example: 

<?xml version="1.0"?>
<VTKFile type="ImageData" version="0.1" byte_order="LittleEndian" compressor="vtkZLibDataCompressor">
  <RD>
    <rule type="formula" name="Schlogl" number_of_chemicals="1" timestep="0.1">
      <formula>
        delta_a = laplacian_a + a - a*a*a;
      </formula>
    </rule>
  </RD>
  <ImageData WholeExtent="0 127 0 63 0 0" Origin="0 0 0" Spacing="1 1 1">
    <Piece Extent="0 127 0 63 0 0">
      <PointData Scalars="Scalars_">
        <DataArray type="Float32" Name="Scalars_" format="appended" RangeMin="-0.0010000000475" RangeMax="0.00099975580815" offset="0" />
      </PointData>
      <CellData>
      </CellData>
    </Piece>
  </ImageData>
  <AppendedData encoding="base64">
   _AQAAAACAAAAAAAAAqXAAAA==eJwkXHtcjPn3n6SbS5KQJGmTJBlJPedsMpIUUhQVRRGSJEmSth1t2 ... lots of data ... 3nudNY=
  </AppendedData>
</VTKFile>

This is the standard VTK format but with an RD element (inside the VTKFile element) that contains all the Ready-specific data. Such files can still be read as standard VTK image format, for example in Paraview. The XML tags are listed below:

The <RD> tag

Specifies the current rule and information about the pattern.
Attributes:
None.
Nested elements:
  • <rule> (required)
  • <pattern_description> (optional)
  • <initial_pattern_generator> (optional)

The <rule> tag

Attributes:
  • type (required) : Either "inbuilt" for inbuilt rules (name must match one) or "formula".
  • name (required) : The name of this rule.
  • number_of_chemicals (required if type="formula") : The number of chemicals in this rule.
  • timestep (required) : The amount of time between each iteration. Smaller values give more accurate results but take longer to compute. Larger values may result in numerical instability.
Nested elements:
  • <description> (optional) : A description of this rule.
  • <param> (multiple, optional) : The parameters of the rule. For inbuilt rules, their names must match those expected. For formula rules, their names should appear in the formula.
  • <formula> (required if rule type="inbuilt") : The differential equation that specifies this reaction-diffusion rule.

The <description> tag

A description of something.
Attributes:
None.
Nested elements:
Plain text.

The <param> tag

Attributes:
  • name (required) : The name of this parameter.
Nested elements:
The value of this parameter.
